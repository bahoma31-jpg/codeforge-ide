## Customizing Shortcuts

### View & Edit Shortcuts

**Open Keyboard Shortcuts Panel:**
- Windows/Linux: `Ctrl+K Ctrl+S`
- macOS: `Cmd+K Cmd+S`

**Or press `?` anywhere in the IDE**

### Custom Keybindings

Create a `keybindings.json` file:

```json
[
  {
    "key": "ctrl+shift+c",
    "command": "git.commit",
    "when": "editorTextFocus"
  },
  {
    "key": "ctrl+shift+p",
    "command": "git.push",
    "when": "!terminalFocus"
  }
]
```

### Conditional Keybindings

```json
[
  {
    "key": "ctrl+enter",
    "command": "editor.action.insertLineAfter",
    "when": "editorTextFocus && !suggestWidgetVisible"
  }
]
```

### Reset to Defaults

```bash
# Reset all keybindings to default
Settings > Keyboard Shortcuts > Reset to Defaults
```

## Tips & Tricks

### Multi-Cursor Editing

```
1. Hold Alt and click
2. Or: Ctrl+Alt+Up/Down
3. Or: Select word and Ctrl+D multiple times
4. Edit all cursors simultaneously
```

### Quick File Navigation

```
1. Press Ctrl+P
2. Type filename (fuzzy search)
3. Press Enter to open
4. Add :line_number to jump to specific line
   Example: "file.ts:42"
```

### Command Palette

```
1. Press Ctrl+Shift+P
2. Type command name
3. See all available actions
4. Execute command directly
```

## Platform-Specific Notes

### Windows
- `Ctrl` is the primary modifier
- `Alt` for secondary actions
- Windows key not commonly used

### macOS
- `Cmd` replaces `Ctrl` in most cases
- `Option` replaces `Alt`
- `Control` used for terminal shortcuts

### Linux
- Similar to Windows
- May vary by desktop environment
- Super key (Windows key) available
